## Findings

### Critical (must fix before proceeding)
1. **Feature branch task missing.** The todo names `feature/subventia-oracle-mvp` as the branch but no task creates it. CLAUDE.md: "Create a new feature branch from the main branch before writing any implementation code." Fix: add T-1b to Block 1 — `git checkout -b feature/subventia-oracle-mvp` — after T-1 (scaffold) and before any code is written.

### Concerns (should address)
2. **T-1 scaffold method is ambiguous.** The repo already contains files (`README.md`, `.claude/`). `npx create-next-app@14 .` will either fail or prompt on a non-empty directory. The task should specify manual scaffolding: create `package.json`, `tailwind.config.js`, `postcss.config.js`, and `app/globals.css` by hand rather than using `create-next-app`. The `nextjs-sqlite` skill defines exactly what's needed — manual setup is safer and more predictable here.

3. **T-4: ANTHROPIC_API_KEY value cannot be auto-generated.** The task says `ANTHROPIC_API_KEY=<value>` but the executor can't know the user's key. The task should note: check whether `.env.local` already exists from a prior session; if not, pause and ask the user for the key before continuing.

4. **T-7 and T-8 describe the same file.** Both tasks are about `app/api/analyze/route.js`. Splitting "write the handler" (T-7) from "embed the prompt" (T-8) creates an awkward partial-done state. Suggest merging into one T-7: "Write complete `app/api/analyze/route.js` — validation, DB query, financial_analysis prompt, Claude call, JSON parse." T-8 can be repurposed as the smoke test (currently T-10), shifting T-9/T-10 up one.

### Suggestions (nice to have)
5. T-10 smoke test should note explicitly: dev server must be running and `.env.local` must have a valid key. This is a prerequisite, not an assumption.
